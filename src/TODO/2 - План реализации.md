# План реализации: Комплексное улучшение системной информации

## Цель
Полная модернизация страницы системной информации (`/info/backend/sys-info`) с:
- ✅ **Архитектурным рефакторингом** на Clean Architecture (Providers → Service → Controller)
- ✅ **Расширением метрик**: Docker, MySQL, Redis, Yii2 (+ текущие системные)
- ✅ **Современным UI**: Bootstrap 5 карточки, вкладки, адаптивность
- ✅ **Графиками**: Chart.js для истории CPU/Memory (60 точек × 3s = 3 мин)
- ✅ **Дополнительными фичами**: логи контейнеров, алерты, экспорт JSON/CSV, настройки

## Требования (от пользователя)
- ✅ **БЕЗ кеширования** → данные всегда свежие (Providers stateless)
- ✅ **БЕЗ jQuery** → только нативный JavaScript (TypeScript + Vite + Fetch API)
- ✅ **С Chart.js** → графики истории метрик
- ✅ **Не потерять информацию** → вся текущая информация сохранена + расширена

## Текущие проблемы
❌ **Нарушение архитектуры**: Controller → Repository напрямую (статические методы)
❌ **Service не используется**: пустой класс, инжектится но не вызывается
❌ **Отсутствие слоя Providers**: нет разделения ответственности
❌ **Устаревший UI**: HTML таблицы вместо Bootstrap 5 карточек
❌ **Самописный JS**: велосипед вместо стандартного Fetch API
❌ **Ограниченные метрики**: только Linux `/proc/*`, нет Docker/DB/App метрик

## 📐 Архитектура решения (Clean Architecture)

### Слои приложения
```
Controller (тонкий слой)
    ↓
Service (бизнес-логика, координация)
    ↓
Providers (сбор метрик из разных источников)
    ↓
Источники (Linux /proc, Docker CLI, MySQL, Redis, Yii2)
```

### 1. Providers Layer (новый слой)

**Структура директории:**
```
Besnovatyj/Info/providers/
├── MetricProviderInterface.php     # Контракт для всех провайдеров
├── BaseMetricProvider.php          # Базовый класс с error handling
├── SystemMetricProvider.php        # CPU, RAM, Disk, Network (/proc/*)
├── PhpMetricProvider.php           # PHP info, OPcache, extensions
├── DockerMetricProvider.php        # Docker контейнеры (shell_exec)
├── DatabaseMetricProvider.php      # MySQL статистика
├── RedisMetricProvider.php         # Redis статистика
└── ApplicationMetricProvider.php   # Yii2 (cache, queue, sessions)
```

**Ключевые методы интерфейса:**
```php
interface MetricProviderInterface {
    public function isAvailable(): bool;      // Проверка доступности источника
    public function getMetrics(): array;      // Получение метрик
    public function getName(): string;        // Имя провайдера
    public function getCategory(): string;    // Категория (system, database, etc.)
}
```

**Решение: НЕ использовать DTOs** (используем типизированные массивы + PHPDoc)

### 2. Service Layer

**SysInfoService** - координирует всех провайдеров:
```php
class SysInfoService {
    public function __construct(
        SystemMetricProvider $system,
        PhpMetricProvider $php,
        DockerMetricProvider $docker,
        DatabaseMetricProvider $database,
        RedisMetricProvider $redis,
        ApplicationMetricProvider $application
    ) { ... }

    public function getAllMetrics(): array              // Все метрики
    public function getRealtimeMetrics(): array         // Только для обновления (легковес)
    public function getMetricsByCategory(string): array // Фильтр по категории
    public function exportToJson(): string              // Экспорт в JSON
    public function exportToCsv(): string               // Экспорт в CSV
    public function getDockerLogs(string, int): array   // Логи контейнера
}
```

### 3. Controller Layer

**SysInfoController** - тонкий слой REST API:
- `actionIndex()` → HTML страница с виджетом
- `actionApiMetrics()` → JSON все метрики
- `actionApiRealtime()` → JSON для обновления
- `actionApiDockerLogs()` → JSON логи контейнера
- `actionExportJson()` → Скачать JSON файл
- `actionExportCsv()` → Скачать CSV файл

### 4. Repository Layer (рефакторинг)

**SystemInfoRepository** - убрать сбор метрик (перенесены в Providers):
- Оставить только для истории метрик (опционально)
- `saveSnapshot()` → сохранить снапшот в БД
- `getHistory()` → получить историю

## 🔧 Edge Cases и обработка ошибок

### 1. Docker недоступен
```php
// DockerMetricProvider::isAvailable() → false
// getMetrics() → ['available' => false, 'message' => 'Docker not available']
// Frontend скрывает Docker вкладку или показывает warning
```

### 2. Redis не настроен
```php
// RedisMetricProvider::checkAvailability()
if (!Yii::$app->has('redis')) return false;
try { Yii::$app->redis->executeCommand('PING'); } catch { return false; }
```

### 3. Shell_exec отключен
```php
// DockerMetricProvider проверяет function_exists('shell_exec')
if (!function_exists('shell_exec')) return false;
```

### 4. Недостаточно прав для /proc
```php
// SystemMetricProvider::checkAvailability()
return is_readable('/proc/stat') && is_readable('/proc/meminfo');
```

### 5. Таймауты команд
```php
// Использовать timeout в shell командах
$output = @shell_exec('timeout 5 docker ps ...');
```

### 6. Большие логи
```php
// Ограничение: max 1000 строк, max 1MB вывода
$lines = min($lines, 1000);
if (strlen($output) > 1024 * 1024) {
    $output = substr($output, -1024 * 1024);
}
```

## 📊 Новые метрики (детально)

### Docker (DockerMetricProvider)
- ✅ Список контейнеров (ID, Name, State, Status, Image)
- ✅ Docker версия
- ✅ Количество images/volumes/networks
- ✅ Логи контейнера (N последних строк)
- ✅ Статистика контейнера (CPU%, Memory, Network I/O)

### MySQL (DatabaseMetricProvider)
- ✅ Тип и версия БД
- ✅ Размер БД в MB
- ✅ Количество таблиц
- ✅ Активные соединения (Threads_connected)

### Redis (RedisMetricProvider)
- ✅ Версия Redis
- ✅ Uptime в секундах
- ✅ Подключенные клиенты
- ✅ Используемая память (used_memory_human)
- ✅ Peak memory
- ✅ Keyspace (db0, db1...)

### Yii2 Application (ApplicationMetricProvider)
- ✅ Environment (dev/prod)
- ✅ Debug mode (on/off)
- ✅ App name
- ✅ Yii версия
- ✅ Queue driver class
- ✅ Cache driver class
- ✅ Session (isActive, ID, name)

### PHP (PhpMetricProvider)
- ✅ Версия PHP
- ✅ SAPI (php-fpm)
- ✅ OPcache (enabled, memory usage, statistics)
- ✅ Memory limit / usage (current, peak)
- ✅ Post max size, upload max filesize
- ✅ Max execution time, socket timeout
- ✅ Загруженные расширения (curl, gd, mbstring, redis, etc.)

### System (SystemMetricProvider) - сохранено всё текущее:
- ✅ Hostname, Server IP, Remote IP
- ✅ OS Distribution, Uname, Kernel
- ✅ Uptime (days, hours, minutes, formatted)
- ✅ CPU (model, cores, frequency, L2 cache, BogoMIPS)
- ✅ CPU Temperature (CPU, GPU if available)
- ✅ CPU Usage (user, sys, nice, idle, iowait, irq, softirq, steal)
- ✅ Memory (total, used, free, buffers, cached, percentages)
- ✅ SWAP (total, used, free, percent)
- ✅ Disk (total, used, free, percent)
- ✅ Load Average (1min, 5min, 15min)
- ✅ Network interfaces (RX/TX bytes formatted, delta realtime)

```javascript
class ChartManager {
    constructor() {
        this.charts = new Map();
        this.maxDataPoints = 60; // 60 секунд истории
    }

    createChart(canvasId, label) {
        const ctx = document.getElementById(canvasId).getContext('2d');
        const chart = new Chart(ctx, {
            type: 'line',
            data: {
                labels: [],
                datasets: [{
                    label: label,
                    data: [],
                    borderColor: 'rgb(75, 192, 192)',
                    tension: 0.1
                }]
            },
            options: {
                responsive: true,
                scales: {
                    y: { beginAtZero: true, max: 100 }
                }
            }
        });

        this.charts.set(canvasId, chart);
    }

    addDataPoint(metrics) {
        this.charts.forEach((chart, id) => {
            const time = new Date().toLocaleTimeString();
            const value = metrics[id]; // Значение метрики

            chart.data.labels.push(time);
            chart.data.datasets[0].data.push(value);

            // Ограничение истории
            if (chart.data.labels.length > this.maxDataPoints) {
                chart.data.labels.shift();
                chart.data.datasets[0].data.shift();
            }

            chart.update('none'); // Без анимации для производительности
        });
    }
}
```

### Система алертов

```javascript
class AlertManager {
    constructor() {
        this.thresholds = {
            cpu: {warning: 70, critical: 85},
            memory: {warning: 75, critical: 90},
            disk: {warning: 80, critical: 95},
        };
        this.notifiedMetrics = new Set();
    }

    checkThresholds(metrics) {
        Object.entries(metrics).forEach(([key, value]) => {
            if (this.thresholds[key]) {
                const threshold = this.thresholds[key];

                if (value >= threshold.critical) {
                    this.showAlert(key, value, 'critical');
                } else if (value >= threshold.warning) {
                    this.showAlert(key, value, 'warning');
                } else {
                    this.notifiedMetrics.delete(key);
                }
            }
        });
    }

    showAlert(metric, value, level) {
        const alertKey = `${metric}-${level}`;
        if (this.notifiedMetrics.has(alertKey)) return;

        this.notifiedMetrics.add(alertKey);

        showAlert({
            message: `${metric.toUpperCase()}: ${value}% (${level})`,
            type: level === 'critical' ? 'error' : 'warning',
            duration: 5000
        });
    }
}
```

```php
            'mysql' => $this->getMysqlStats(),
            'redis' => $this->getRedisStats(),

        $db = Yii::$app->db;

            'version' => $db->createCommand('SELECT VERSION()')->queryScalar(),
            'connections' => $db->createCommand('SHOW STATUS LIKE "Threads_connected"')->queryOne(),

            'used_memory' => $redis->executeCommand('INFO', ['memory']),
            'keys' => $redis->executeCommand('DBSIZE'),
            'hit_rate' => $this->calculateHitRate($info),
        ];

}
```
